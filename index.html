1. Project Overview

What is a VPN?

A Virtual Private Network (VPN) creates a secure, encrypted tunnel over a public network like the Internet. It lets remote users connect to a private network as if they were physically there, ensuring confidentiality, integrity, and authentication of all data.

  • Secure remote access for employees working from home
  • Connecting multiple office branches (site-to-site VPN)
  • Protecting data transmitted over untrusted networks
  • Remote management of network infrastructure

What is pfSense?

pfSense is a free, open-source firewall/router based on FreeBSD, developed by Netgate. It's widely used in enterprise, education, and home lab environments.

  • Stateful packet inspection firewall
  • Supports OpenVPN, IPsec, WireGuard, L2TP
  • Web-based GUI management (WebConfigurator) via HTTPS
  • Runs on physical hardware or VMs (VMware, VirtualBox, Hyper-V)

3. Implementation Steps

Step 1 — Install pfSense on VMware

  1. Download pfSense 2.7.2 ISO from pfsense.org/download
  2. Create VM: 2 vCPUs, 2 GB RAM, 20 GB disk, 64-bit FreeBSD
  3. Add Network Adapter 1 → NAT (VMnet8) and Adapter 2 → Host-Only (VMnet1)
  4. Boot from ISO, follow installer, assign WAN=em0 / LAN=em1 from console

Step 2 — Configure WAN & LAN

  1. WAN (em0): set to DHCP → auto-receives 192.168.186.134/24
  2. LAN (em1): set static IP 192.168.1.1, subnet /24

Step 3 — Access pfSense WebGUI

  1. Set Windows PC VMnet1: IP=192.168.1.100, GW=192.168.1.1, DNS=8.8.8.8
  2. Open browser → https://192.168.1.1 → accept cert → login: admin / pfsense
  3. Run Setup Wizard: hostname, DNS 8.8.8.8/8.8.4.4, timezone Asia/Phnom_Penh, change password

Step 4 — Set Up OpenVPN Server

  1. Go to VPN → OpenVPN → Wizards → Authentication: Local User Access
  2. Create CA: name = pfSense-CA, key length 2048
  3. Server settings: Protocol = UDP, Interface = WAN, Port = 1194
  4. Tunnel Network: 10.0.8.0/24 | Local Network: 192.168.1.0/24
  5. Enable Redirect Gateway + DNS — wizard auto-creates firewall rules

Step 5 — Create User & Export Config

  1. System → User Manager → Add: set username/password, create user certificate (vpnuser-cert)
  2. Install package: System → Package Manager → openvpn-client-export
  3. VPN → OpenVPN → Client Export → download .ovpn file
  4. Install OpenVPN Connect, import the .ovpn, connect, verify IP in 10.0.8.x range

6. Benefits & Drawbacks

Benefits

  • AES-256-CBC encryption protects all traffic on public networks
  • Remote access to LAN resources (192.168.1.0/24) from anywhere
  • Centralized management via pfSense WebGUI
  • Completely free — no per-user licensing fees (Community Edition)
  • Cross-platform clients: Windows, macOS, Linux, Android, iOS
  • Built-in session monitoring at Status → OpenVPN

Drawbacks

  • Encryption adds CPU overhead, reducing maximum throughput
  • Setup complexity — CA, certificates, and firewall rules require networking knowledge
  • Single point of failure without HA (CARP) setup
  • No official support in CE — relies on community forums
  • SSL certificates expire and must be renewed manually

8. Conclusion

This project successfully deployed a functional VPN server using pfSense 2.7.2 on VMware Workstation. The setup used OpenVPN with AES-256 encryption, certificate-based authentication, and cross-platform client support — all at no cost using the Community Edition.

pfSense proves to be a solid, flexible solution for both learning environments and real-world deployments, combining a powerful FreeBSD foundation with an easy-to-use web interface.
